# Sensitive Guard - FUSE 方案成本分析

## 安装成本

### 1. macFUSE
- **大小**: 14 MB
- **安装方式**: `brew install --cask macfuse`
- **耗时**: 约 30-60 秒（已有 Homebrew）
- **权限**: 需要授予系统扩展权限（首次安装需重启）
- **用户操作**:
  1. 运行安装命令
  2. 系统弹窗 → 打开"系统偏好设置"
  3. 安全性与隐私 → 允许 macFUSE
  4. **需要重启 Mac**（首次安装）

### 2. Python 依赖
- **大小**: 13 MB（虚拟环境）
- **安装方式**: 自动创建虚拟环境 + pip install fusepy
- **耗时**: 约 10-20 秒
- **权限**: 无需特殊权限
- **用户操作**: 无（安装脚本自动处理）

### 3. Sensitive Guard 本体
- **大小**: < 1 MB
- **文件**: Python 脚本 + Shell 包装器
- **耗时**: < 5 秒
- **权限**: 无需特殊权限

---

## 总成本

| 项目 | 大小 | 耗时 | 用户操作 |
|------|------|------|---------|
| macFUSE | 14 MB | 30-60s | 需授权 + 重启 |
| Python 环境 | 13 MB | 10-20s | 无 |
| 本体 | < 1 MB | < 5s | 无 |
| **总计** | **~28 MB** | **1-2 分钟** | **首次需重启** |

---

## 关键问题

### ⚠️ 用户感知严重的问题

1. **首次安装需要重启 Mac**
   - macFUSE 是内核扩展，必须重启才能生效
   - 这对演示场景来说体验很差

2. **系统权限授予流程复杂**
   - 需要打开系统偏好设置
   - 需要点击"允许"按钮
   - 可能被安全软件拦截

3. **依赖外部组件**
   - macFUSE 不是系统自带
   - 如果 macFUSE 更新可能导致兼容性问题

---

## 建议

**对于演示工具来说，FUSE 方案的用户成本太高：**
- ❌ 需要重启（演示前临时安装不现实）
- ❌ 权限授予流程复杂（用户可能不会操作）
- ❌ 依赖外部组件（增加维护成本）

**推荐回到轻量方案：**
- ✅ 零依赖（纯 Shell）
- ✅ 零权限（不需要系统授权）
- ✅ 零重启（即装即用）
- ✅ 覆盖终端场景（演示主要场景）
